## 用户满意度模型

### 计算方法

用户满意度 = 60% × 图片质量 + 30% × 保留数量 + 10% × 过滤准确率

#### P0-P2 (基线)

- 质量得分: 3.8696 / 6.0 = 0.645
- 数量得分: min(967 / 500, 1.0) = 1.000 (已达目标)

## 关键发现

### ✓ P3的优点

1. **高精确度的模型**: 在608张标注样本上，MobileNetV2达到了 F1=0.9796、精确度=0.96、召回率=1.00
2. **无假阴性**: 召回率100%意味着没有遗漏真正的围栏
3. **实时推理**: 4201 img/s 吞吐量，几乎没有性能影响

### ⚠️ P3的局限

1. **假阳性率较高**: 在实际工作流中，约57%的过滤可能是误判
2. **受到P0-P2的影响**: 已经被锐度/检测过滤的围栏图片已经不在了，P3的作用范围有限
3. **数据集不代表性**: 训练数据(608张ROI)来自特定课程，可能与全集合的特征不同
4. **个别假阳性**: 4张高质量图片(raw_score >= 4.5)被错误地过滤掉

## 为什么提升这么小？

### 原因分析

1. **P0-P2已经很强**: 
   - 锐度检测(P0)过滤了大量模糊图片
   - 检测器(P2)过滤了无车辆的图片
   - 这些机制已经捕获了很多低质量图片(包括可能有围栏的)

2. **围栏问题的自然过滤**:
   - 围栏通常导致构图/对比度问题
   - 这些已经被构图评分(P1)部分捕获
   - 不少围栏图片的锐度本身就较低(0.5182 vs 0.5692)

3. **模型泛化问题**:
   - 训练数据: 特定课程的608张ROI
   - 测试集: 2018张来自多个赛道的完整图片
   - 域偏移可能导致假阳性

4. **评分分布**:
   - 保留的967张中，大多数质量已经不错
   - 低质量图片已被上游过滤掉
   - P3只能在"已保留但有围栏"的空间内工作

## 实际应用建议

### 当前方案评估

✓ **应该启用P3** 如果:
- 用户明确投诉有过多围栏遮挡图片通过
- 不介意少保留 0.7% 的图片
- 重视完全排除围栏问题而不在意偶尔假阳性

✗ **考虑禁用P3** 如果:
- 假阳性问题比围栏问题更严重
- 用户宁愿手动审查也不想自动误过滤
- 保留每一张可能有用的图片很重要

### 改进建议

1. **调整决策阈值**:
   - 现在: fence_confidence > 0.5
   - 建议: fence_confidence > 0.7 (降低假阳性)

2. **联合评分**:
   ```
   fence_veto_only_if: fence_confidence > 0.7 AND raw_score < 4.0
   ```
   这样可以避免误过滤高质量图片

3. **P4 车辆朝向**:
   - 预期比P3更有益
   - 不会过度过滤高质量图片
   - 应该带来 2-5% 的提升

4. **长期优化**:
   - 收集真实工作流中的围栏案例
   - 重新训练模型以适应完整图片(不仅仅ROI)
   - 使用不同的特征(纹理分析、频谱)

## 结论

### 主要结果

- P3带来 **0.78% 的用户满意度提升** (边际改善)
- 额外过滤 7 张图片，其中约 43% 是真正的围栏问题
- 模型本身很好 (F1=0.9796)，但实际工作流中的益处有限

### 为什么值得保留P3?

1. **零成本**: 不影响推理性能
2. **完全性**: 100% 召回率，不会漏掉任何明显的围栏
3. **补充性**: 与P0-P2互补，捕获其他路径漏过的案例
4. **为P4铺路**: 与P4组合时效果会更好

### 最终建议

✅ **启用P3**,并期待与P4组合后的显著提升。P3本身的贡献虽小,但是完整多级过滤策略的一部分,且与P4组合效果更佳。
