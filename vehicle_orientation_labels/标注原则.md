# 车辆朝向标注指南

## 标注原则

本任务是对 F1 赛车及其他赛车的朝向（Heading/Orientation）进行 4 分类。

### 4 个朝向类别定义

#### 1. **正前方** (0°)
- 车头正对向画面（或接近正对）
- 看不到或极少看到侧面
- 能清晰看到前翼、方向盘区域、前保险杠
- **特征**：车身最窄，对称性最强

#### 2. **侧身** (90°)
- 车身完全横向，正侧面对画面
- 能清晰看到整个车侧（驾驶舱、侧翼、底盘线条）
- 看不到车头或车尾
- **特征**：车身最宽，可见侧面完整

#### 3. **正后方** (180°)
- 车尾正对向画面
- 能清晰看到后翼、排气、后保险杠
- 镜像于"正前方"
- **特征**：与正前方对称，车尾最明显

#### 4. **侧后方** (225° 或 135°)
- 车身45°角，介于"侧身"和"正后方"之间
- 能同时看到车的侧面和部分车尾/车头
- 通常是比赛时超车、防守、过弯时的典型角度
- **特征**：侧面和前/后部都可见

### 标注注意事项

1. **以车身主轴为准**：不要被摄像机角度（俯视、仰视）迷惑，判断车的前后左右
2. **边界情况处理**：
   - 如果难以判断"侧身"和"侧后方"，倾向选择更接近的类别
   - 极端侧后方（>170° 偏后）应归为"正后方"
   - 极端侧前方（<10° 偏前）应归为"正前方"
3. **只看车，不看背景**：忽略赛道、观众、其他物体
4. **清晰度要求**：即使车在 ROI 中占比小（3-5%），只要能判断朝向就可以标注

### 如果无法判断

- ROI 过于模糊或车身被部分遮挡，无法确定朝向 → **删除这个文件**
- ROI 中没有车或仅有人像 → **删除**（应该已在之前筛选中清除）

## 文件夹结构

```
vehicle_orientation_labels/
├── 正前方/           ← 车头对着摄像机
├── 侧身/             ← 车身 90° 侧向
├── 正后方/           ← 车尾对着摄像机
└── 侧后方/           ← 车身 45° 后方向
```

## 工作流

1. 从 `fence_label/有铁丝网/` 和 `fence_label/没有铁丝网/` 中复制所有 JPG
2. 对每一张，判断车的朝向
3. 移动到对应子文件夹
4. 无法判断的删除

---

**估计工作量**：608 张 × 1-2 分钟/张 ≈ 10-20 小时

**建议节奏**：分批处理，每批 100 张后休息
